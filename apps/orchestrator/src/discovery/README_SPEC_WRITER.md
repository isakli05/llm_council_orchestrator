# DomainSpecWriter Usage Guide

## Overview

The `DomainSpecWriter` generates YAML specification files for discovered domains. Each DEEP domain gets its own spec file containing discovery metadata, confidence scores, signals, and supporting evidence.

## Basic Usage

```typescript
import { DomainSpecWriter } from './DomainSpecWriter';
import { Domain } from './types';

// Configure the writer
const writer = new DomainSpecWriter({
  outputDir: './output/specs',
  createDir: true, // Optional: create directory if it doesn't exist
});

// Write a single domain spec
const result = await writer.writeDomainSpec(domain);

if (result.success) {
  console.log(`Spec written to: ${result.filePath}`);
} else {
  console.error(`Failed: ${result.error?.message}`);
}

// Write multiple domains at once
const results = await writer.writeDomainSpecs(domains);
```

## Generated Spec Format

Each spec file follows this YAML structure:

```yaml
# Domain Specification
# Generated by Domain Discovery Engine (RAA)

spec_version: "1.0.0"
generated_at: "2025-12-12T14:29:03.187Z"

discovery_metadata:
  analysis_depth: "DEEP"
  confidence: 0.870

domain:
  id: "auth_domain"
  name: "Authentication"
  confidence: 0.870

signals:
  file_pattern:
    - value: "/auth"
      weight: 0.900
      source: "directory_structure"
  dependency:
    - value: "passport"
      weight: 0.900
      source: "package.json"

evidence:
  - file_path: "src/auth/AuthController.ts"
    relevance_score: 0.950
    line_range:
      start: 1
      end: 50
    snippet: "export class AuthController { login() { ... } }"

sub_domains:
  - id: "auth_api_domain"
    name: "Auth API"
    confidence: 0.850
    analysis_depth: "DEEP"
```

## Key Features

### 1. DEEP Domain Filtering
Only domains with `analysisDepth = "DEEP"` can have specs generated. Attempting to write a spec for an EXCLUDED domain returns an error.

### 2. Signal Grouping
Signals are automatically grouped by type (file_pattern, dependency, framework, route, config) for better organization.

### 3. Special Character Escaping
The writer automatically escapes special characters in YAML strings:
- Quotes: `"` → `\"`
- Backslashes: `\` → `\\`
- Newlines: `\n` → `\\n`
- Tabs: `\t` → `\\t`

### 4. Optional Fields
- `lineRange`: Only included if present in evidence
- `snippet`: Only included if present in evidence
- `sub_domains`: Only included if domain has sub-domains

### 5. Batch Processing
The `writeDomainSpecs()` method:
- Filters to DEEP domains automatically
- Continues processing if one domain fails
- Returns results for all domains

## File Naming

Spec files are named using the domain ID:
- Domain ID: `auth_domain` → File: `auth_domain.yaml`
- Domain ID: `payment_domain` → File: `payment_domain.yaml`

## Error Handling

The writer handles several error cases:

1. **Non-DEEP Domain**: Returns error with code `INVALID_DOMAIN_DEPTH`
2. **File System Errors**: Returns error with code `SPEC_WRITE_ERROR`
3. **Directory Creation**: Automatically creates output directory if `createDir: true`

## Integration with Pipeline

The DomainSpecWriter is designed to be used in the SPECIFY pipeline state:

```typescript
// In PipelineEngine or spec generation step
const writer = new DomainSpecWriter({
  outputDir: path.join(process.cwd(), 'output', 'domain-specs'),
});

const discoveryResult = context.discoveryResult;
const results = await writer.writeDomainSpecs(discoveryResult.domains);

// Log results
const successCount = results.filter(r => r.success).length;
logger.info('Domain specs generated', { successCount });
```
